# 个人知识工作台 PRD

## 1. 产品定义

个人知识工作台是一个本地运行、聊天优先、单用户的知识工作产品。

产品核心不是泛聊天，而是围绕个人项目资料完成可追溯问答、深度调研与结果沉淀。

## 2. 目标用户

面向普通知识工作者，尤其适合以下场景：

- 围绕长期主题持续收集网页、PDF、DOCX 等资料
- 在自己的项目资料范围内提问和追问
- 在复杂问题上升级为深度调研
- 将高价值结果沉淀为摘要或报告

## 3. 核心问题

用户真正的困难不是缺一个聊天框，而是：

- 资料分散，缺少项目级知识容器
- 提问后很难稳定基于自有资料回答
- 回答缺少来源边界，难以复核
- 复杂问题需要人工多步整理，成本高
- 高价值结论难以沉淀和复用

## 4. 核心价值

v1-v6 阶段聚焦三件事：

1. 以项目为容器组织资料
2. 围绕资料进行可追溯问答
3. 在复杂问题上支持深度调研，并把结果沉淀为摘要或报告

## 5. 核心对象

- 项目：知识容器，承载资料与会话
- 会话：项目内的对话线程
- 消息：用户问题、助手回答、状态卡、结果卡
- 资料：文件或网页等知识输入
- 来源：回答底部可展开的轻量证据层
- 摘要卡：沉淀简洁结论
- 报告卡：沉淀结构化输出

## 6. 主闭环

绝对主闭环：

`创建项目 -> 新建会话 -> 增加资料 -> 项目内提问 -> 查看来源`

扩展闭环：

`复杂提问 -> 打开深度调研 -> 保存为摘要 / 生成报告`

联网补充是辅助能力，不改变“项目优先”的产品定位：

- 项目证据足够时，答案以项目资料为主
- 项目证据不足但联网补充足够时，允许基于网页补充形成回答
- 两侧证据都不足时，应明确说明无法提供完整结论

## 7. 用户可见能力

### 7.1 项目

- 创建项目
- 浏览最近活跃项目
- 搜索项目
- 归档项目

### 7.2 会话

- 在项目内新建会话
- 按项目浏览会话
- 自动生成标题
- 手动重命名和删除会话

### 7.3 聊天

- 基于项目资料提问
- 在同一会话内持续追问
- 打开深度调研
- 选择是否启用联网补充
- 生成报告
- 保存为摘要

### 7.4 资料与来源

- 从会话内或知识库页增加资料
- 支持文件与网页导入
- 在回答底部查看来源列表与详细预览

### 7.5 设置

- 在 `/settings` 统一配置全局模型、向量模型、重排模型与 API Key
- 设置页只负责系统配置，不承载项目或知识库操作

## 8. 信息架构

公开路由固定为：

- `/workspace`
- `/sessions`
- `/knowledge`
- `/settings`
- `/projects/[projectId]`
- `/projects/[projectId]?sessionId=...`

其中：

- `/workspace` 是唯一项目入口页
- `/sessions` 是全局会话浏览页，不是主工作区
- `/knowledge` 是资料管理页
- `/projects/[projectId]` 是主工作区

## 9. 非目标

以下内容不属于当前产品方向：

- task-first 工作台
- 多人协作、认证、workspace 体系
- 独立 asset center
- 独立 search page
- 通用 Agent 平台叙事

## 10. 前台术语

前台统一使用：

- 项目
- 会话
- 消息
- 知识库
- 来源
- 深度调研
- 联网补充
- 保存为摘要
- 生成报告

不要在前台主文案使用：

- task
- asset
- task detail

## 11. 成功标准

V6 的成功标准不是继续堆功能，而是：

- 项目聊天主链稳定
- 来源层语义一致
- 深度调研与联网补充协同清晰
- 摘要卡与报告卡继续留在会话中
- 设置页成为可信的全局配置入口
