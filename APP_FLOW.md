# 个人知识工作台 使用流程

## 1. 目的

本文定义聊天优先 v1 的用户主流程。

公开路由：

- `/workspace`
- `/sessions`
- `/knowledge`
- `/settings`
- `/projects/[projectId]`
- `/projects/[projectId]?sessionId=...`

已移除路由：

- `/tasks`
- `/search`
- `/assets`
- 独立 task 详情页

## 2. 页面清单

### 2.1 `/workspace`

用途：

- 创建新项目
- 打开最近使用的项目
- 按名称搜索项目
- 删除项目（软删除，项目卡片右上角 × 按钮）
- 显示/隐藏已归档项目（可选筛选）

### 2.2 `/sessions`

用途：

- 按项目分组浏览会话
- 进入某个项目内的具体会话
- 重命名或删除会话

### 2.3 `/knowledge`

用途：

- 按项目分组浏览资料
- 搜索资料
- 预览资料
- 从资料管理进入聊天

### 2.4 `/settings`

用途：

- 统一配置大模型、向量模型和重排模型
- 显示当前全局模型配置是否已生效
- 不承载项目、会话、知识库操作

### 2.5 `/projects/[projectId]`

用途：

- 项目空状态页
- 项目知识与会话的主入口
- 未指定 `sessionId` 时不自动打开会话

### 2.6 `/projects/[projectId]?sessionId=...`

用途：

- 在项目聊天页中打开某个具体会话

## 3. 全局规则

- `工作台` 是唯一项目入口页
- `会话` 是全局浏览页，不是主工作面
- `知识库` 负责资料管理，不取代项目聊天页
- 项目聊天页是主工作面
- 用户在项目内工作，而不是在 task 详情页里工作

## 4. 流程 1：打开应用

触发：

- 用户打开应用

系统行为：

1. 默认进入 `/workspace`
2. 显示创建项目入口
3. 如果没有项目，项目列表为空
4. 如果已有项目，展示最近活跃项目列表

用户看到的结果：

- 用户知道工作从“创建项目”或“打开项目”开始

失败与边界：

- 如果项目加载失败，保留创建项目入口并显示重试提示

## 5. 流程 2：创建项目

触发：

- 用户在 `/workspace` 提交创建项目表单

系统行为：

1. 校验名称和描述
2. 创建项目
3. 跳转到 `/projects/[projectId]`
4. 以项目空状态打开

用户看到的结果：

- 用户进入项目页，但还没有自动打开任何会话

失败与边界：

- 如果名称或描述为空，显示行内校验错误
- 如果创建失败，保留表单内容并给出重试提示

## 6. 流程 3：进入项目空状态

触发：

- 用户进入 `/projects/[projectId]` 且没有 `sessionId`

系统行为：

1. 左侧 sidebar 加载项目树
2. 当前项目默认展开
3. 中间区域不自动打开任何会话
4. 中间区域显示项目空状态
5. 空状态至少提供：
   - 新建会话
   - 前往知识库

用户看到的结果：

- 用户明确决定先开始对话，还是先管理资料

失败与边界：

- 如果项目不存在，显示 not found

## 7. 流程 4：新建会话

触发：

- 用户在项目页点击“新建会话”

系统行为：

1. 在当前项目下创建会话
2. 跳转到 `/projects/[projectId]?sessionId=...`
3. 显示聊天输入区
4. 如果当前项目还没有可检索资料，显示弱资料模式提示

用户看到的结果：

- 用户可以立刻开始聊天

失败与边界：

- 如果会话创建失败，停留在项目页并显示重试提示

## 8. 流程 5：在会话内添加资料

触发：

- 用户在聊天输入区点击 `增加资料`

系统行为：

1. 打开轻量菜单
2. 菜单提供：
   - 添加文件
   - 添加网页链接
3. 选择文件时打开文件选择器
4. 选择网页时在输入区上方打开轻量 URL 表单
5. 导入成功后：
   - 当前项目来源列表更新
   - 同一会话中插入一条 `source update` 系统卡片

用户看到的结果：

- 资料进入当前项目
- 用户无需离开会话即可继续追问新资料

失败与边界：

- 入库失败时要明确暴露给用户

## 9. 流程 6：在同一会话里提问

触发：

- 用户在项目会话中输入并发送问题

系统行为：

1. 追加用户消息
2. 如果这是首条用户消息，自动生成会话标题
3. 对当前项目知识执行检索
4. 如果有证据，基于小规模最终证据集生成 grounded 回答
5. 助手消息壳先出现，随后流式写入回答
6. 回答完成后在底部挂载最终来源气泡
7. 如果没有项目证据，则切换到 `weak_source_mode`，但仍保持正常对话流

用户看到的结果：

- 回答出现在同一条对话流里
- 第一条用户消息后，会话标题变得有意义
- 可以继续追问，不会跳到其他页面

失败与边界：

- 如果回答失败，保留用户消息，并显示助手侧失败状态

## 10. 流程 7：开启深度调研

触发：

- 用户在发送前手动打开 `深度调研`

系统行为：

1. 用户仍停留在同一会话
2. 系统插入轻量状态卡，例如：
   - `调研中`
   - `调研完成`
3. 本轮会走更深的证据选择与回答路径
4. 最终回答仍流式进入同一条消息流

用户看到的结果：

- 会话是连续的，即使问题更复杂也不跳出当前线程

失败与边界：

- 如果调研失败，也必须留在同一会话并给出可恢复的失败提示

## 11. 流程 8：查看来源

触发：

- 用户点击某条助手回答下方的来源气泡

系统行为：

1. 先展开轻量的来源标题列表
2. 再点击某个来源标题时
3. 打开覆盖式来源预览面板

用户看到的结果：

- 用户可以在不离开会话的前提下检查证据

失败与边界：

- 如果预览加载失败，保留当前消息上下文并显示重试入口

## 12. 流程 9：保存为摘要或生成报告

触发：

- 用户在当前会话里点击：
  - `保存为摘要`
  - `生成报告`

系统行为：

1. 使用当前会话最近一次有效结论
2. 在同一会话内追加摘要卡或报告卡
3. 原回答保持不变

用户看到的结果：

- 用户在不离开会话的前提下得到可复用结果

## 13. 流程 10：使用知识库页面

触发：

- 用户进入 `/knowledge`
- 或在项目聊天页通过知识库入口进入资料管理

系统行为：

1. 页面按项目加载来源分组
2. 用户可以搜索或按项目过滤
3. 用户先预览来源
4. 用户可点击 `进入聊天`
5. 系统在该项目下创建一个新会话
6. 跳转到 `/projects/[projectId]?sessionId=...`

用户看到的结果：

- 用户可以从资料管理顺滑进入项目对话，而不丢失项目上下文

## 14. 流程 11：删除项目

触发：

- 用户在 `/workspace` 点击项目卡片右上角的删除按钮

系统行为：

1. 弹出确认对话框，提示删除后不可恢复
2. 用户确认后，执行软删除（status 变为 'archived'）
3. 项目从工作台列表中消失
4. 已归档项目的会话和来源不再出现在全局列表中
5. 数据保留 30 天，之后由 CleanupService 自动物理删除

用户看到的结果：

- 项目从工作台消失
- 关联的会话和来源不再出现在列表中
- 无需手动清理，系统自动处理过期数据

失败与边界：

- 删除失败时显示错误提示，项目保持原状态
- 正在删除中的项目显示 loading 状态防止重复操作

## 15. 完成定义

满足以下条件时，v1 主流程成立：

- 用户能从 `工作台` 创建并打开项目
- 用户能在项目页新建会话
- 用户能在会话内或知识库中添加资料
- 用户能在同一会话里提问、查看来源、继续追问
- 用户能在同一会话里保存摘要卡、生成报告卡
- 全流程不依赖旧的 task / search / asset 心智
