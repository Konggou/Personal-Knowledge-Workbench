# 个人知识工作台使用流程

## 1. 目的

本文定义用户在聊天优先产品中的主路径、关键状态与失败边界。

产品公开路由与对象定义以 `PRD.md` 为准；本文只描述“用户如何走完整个闭环”。

## 2. 全局规则

- 主工作区始终是项目聊天页
- 用户在项目内工作，而不是在 task 详情页工作
- 摘要卡和报告卡必须继续留在当前会话中
- 来源层只展示最终证据集，不暴露内部检索过程
- 深度调研和联网补充是会话内提问选项，不产生独立页面

## 3. 页面职责

### 3.1 `/workspace`

用途：

- 创建项目
- 打开最近使用的项目
- 搜索项目
- 查看归档项目

### 3.2 `/sessions`

用途：

- 按项目分组浏览会话
- 进入某个项目中的具体会话
- 重命名或删除会话

### 3.3 `/knowledge`

用途：

- 按项目浏览资料
- 搜索资料
- 预览资料
- 从资料管理跳回项目聊天

### 3.4 `/settings`

用途：

- 查看和保存全局模型、向量模型、重排模型与 API Key
- 清空或更新 API Key
- 说明配置何时生效

### 3.5 `/projects/[projectId]`

用途：

- 作为项目主工作区
- 显示项目空状态
- 展示项目内会话与聊天内容

## 4. 流程 1：打开应用

触发：

- 用户打开应用

系统行为：

1. 默认进入 `/workspace`
2. 显示创建项目入口和最近活跃项目

用户结果：

- 用户知道可以“创建项目”或“进入已有项目”

失败边界：

- 项目加载失败时，仍保留创建项目入口并提示重试

## 5. 流程 2：创建项目

触发：

- 用户在 `/workspace` 提交创建项目表单

系统行为：

1. 校验名称与描述
2. 创建项目
3. 跳转到 `/projects/[projectId]`

用户结果：

- 进入项目页，但不会自动打开某个会话

失败边界：

- 表单非法时显示行内校验
- 创建失败时保留输入内容并给出重试提示

## 6. 流程 3：进入项目空状态

触发：

- 用户进入 `/projects/[projectId]` 且没有 `sessionId`

系统行为：

1. 左侧加载当前项目的会话树
2. 中间区域显示项目空状态
3. 空状态至少提供：
   - 新建会话
   - 前往知识库

用户结果：

- 决定先开始对话还是先补资料

失败边界：

- 项目不存在时显示 not found

## 7. 流程 4：新建会话

触发：

- 用户点击“新建会话”

系统行为：

1. 在当前项目下创建会话
2. 跳转到 `/projects/[projectId]?sessionId=...`
3. 显示聊天输入区
4. 若当前项目暂无可检索资料，显示弱资料模式提示

用户结果：

- 可以立刻开始聊天，不必先离开页面

失败边界：

- 会话创建失败时停留在当前页并显示重试提示

## 8. 流程 5：增加资料

触发：

- 用户在会话内点击“增加资料”
- 或用户在 `/knowledge` 中增加资料

系统行为：

1. 提供文件导入或网页链接导入
2. 资料入库成功后刷新项目资料状态
3. 若从会话内导入，插入一条轻量 `source update` 系统卡

用户结果：

- 新资料立即进入当前项目的检索范围

失败边界：

- 入库失败时应清楚暴露失败原因

## 9. 流程 6：普通提问

触发：

- 用户在项目会话中输入并发送消息

系统行为：

1. 追加用户消息
2. 首条用户消息后自动生成会话标题
3. 对当前项目资料执行 grounded 检索
4. 若命中有效证据，则流式生成答案
5. 回答完成后挂载最终来源气泡
6. 若项目资料偏弱，则进入弱资料模式提示，但仍允许继续对话

用户结果：

- 结果留在同一会话流中，可继续追问

失败边界：

- 回答失败时保留用户消息，并在助手侧显示可恢复的失败状态

## 10. 流程 7：深度调研

触发：

- 用户在发送前打开“深度调研”

系统行为：

1. 本轮提问进入研究型链路
2. 在同一会话中插入轻量状态卡，例如“调研中”“调研完成”
3. 使用更深的证据选择与生成路径
4. 最终回答仍流式进入同一消息流

用户结果：

- 更复杂的问题也不会跳出当前会话

失败边界：

- 调研失败时，也必须留在当前会话并给出可恢复提示

## 11. 流程 8：联网补充

触发：

- 用户在发送前打开“联网补充”

系统行为：

1. 优先检索当前项目资料
2. 仅在项目证据不足或需要补强时引入外部网页证据
3. 如果最终主要依赖网页证据，应在回答中明确说明“主要基于联网补充来源”
4. 若项目资料与网页证据都不足，则明确说明无法提供完整结论

用户结果：

- 用户能理解答案到底是基于项目资料、网页补充，还是证据不足

失败边界：

- 联网能力失败时，应保留项目内回答或明确给出补充失败提示，不得伪装成完全成功

## 12. 流程 9：查看来源

触发：

- 用户点击回答下方来源气泡

系统行为：

1. 先展开来源标题列表
2. 再点击标题进入覆盖式详细预览
3. 支持从详细预览返回来源列表

用户结果：

- 用户在不离开会话的前提下检查证据

失败边界：

- 来源缺失时，应明确说明当前回答未形成可展示来源

## 13. 流程 10：保存为摘要 / 生成报告

触发：

- 用户在有效回答后点击“保存为摘要”或“生成报告”

系统行为：

1. 基于当前会话最近一次有效结论生成结果卡
2. 结果卡插入当前会话流
3. 结果卡继承与其对应结论一致的最终来源集

用户结果：

- 摘要和报告成为会话中的可追溯结果，而不是跳到新页面

失败边界：

- 若缺少有效结论或证据不足，应提示无法生成，而不是创建空卡

## 14. 流程 11：删除

### 14.1 删除会话

触发：

- 用户在会话列表中删除会话

系统行为：

1. 执行软删除
2. 当前项目与全局列表同步更新

用户结果：

- 会话从主视图移除

### 14.2 删除项目

触发：

- 用户删除项目

系统行为：

1. 将项目标记为归档或软删除状态
2. 在清理窗口到期前不立即物理删除
3. 到期后由清理服务清除数据

用户结果：

- 项目对前台不再可见，但删除不是即时不可恢复

## 15. 文档边界

- 产品定义、核心对象、非目标：`PRD.md`
- 页面布局、组件交互、视觉规则：`FRONTEND_GUIDELINES.md`
- 后端结构、模式、API 与存储：`BACKEND_STRUCTURE.md`
- 技术栈与运行方法：`TECH_STACK.md`
