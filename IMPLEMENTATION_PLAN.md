# 实施计划

## 1. 目的

本文只保留当前阶段与下一阶段的实施计划。

已完成历史、阶段复盘与长期演进记录统一写入 `progress.txt`，避免计划文档继续混入历史档案。

## 2. 当前版本定位

当前主线是 V6：聊天主闭环收敛版。

目标不是继续堆新能力，而是让以下闭环更稳：

`项目内提问 -> 流式回答 -> 查看来源 -> 保存为摘要 / 生成报告`

## 3. 当前优先级

### P0 稳定性

- 保持项目聊天页数据流只围绕当前项目会话
- 继续巩固来源层与详细预览闭环
- 维持深度调研与联网补充的证据语义一致
- 保障设置页保存、校验与生效说明稳定

### P1 体验打磨

- 优化项目页切换、重命名、删除后的状态一致性
- 优化弱资料模式与证据不足文案
- 继续清理前台残余乱码和术语漂移

### P2 质量治理

- 补强主闭环前后端回归测试
- 维持 benchmark 默认值和运行时默认值同步
- 继续收敛旧命名残留和重复 helper

## 4. 当前工作包

### 4.1 聊天主链

- 保持项目页聊天组件分层清晰
- 防止全局 session groups 再次侵入项目主工作区
- 维持会话级开关状态记忆

### 4.2 来源与证据

- 统一回答、摘要卡、报告卡的来源挂载规则
- 保持来源只代表最终证据集
- 明确区分 `project_grounded`、`web_supplement_grounded`、`insufficient`

### 4.3 设置与评测

- 保持 `/settings` 为全局配置页
- 保持 benchmark 仅作为评测与默认值校准工具
- 继续锁住默认参数一致性测试

## 5. 下一阶段候选项

按推荐顺序：

1. 继续打磨深度调研与联网补充的边界话术
2. 把会话级提问开关持久化到后端
3. 清理高频文档与源码中的编码异常
4. 继续削减旧 task-like 内部命名债

## 6. 明确不做

当前阶段不做：

- 新增 task dashboard
- 新增 asset center
- 新增独立 search page
- 引入多人协作、认证、workspace
- 把产品叙事改成 Agent 平台

## 7. 验收标准

- 项目聊天主链可稳定回归
- 来源层交互闭环一致
- 深度调研与联网补充不再语义打架
- 摘要卡与报告卡继续留在会话中
- 设置页与 benchmark 默认值保持同步
