Known Gaps
- Retrieval quality is solid but still basic compared with a more advanced semantic chunking + rerank pipeline
- Embedded Qdrant is validated; remote Qdrant server mode still deserves a separate end-to-end validation pass
- Session event stream is currently replay-oriented, not token-by-token live generation
- Legacy `.doc` parsing is still not implemented; the app now surfaces this clearly and supports `.pdf` / `.docx`

V1 Simplifications / Later Improvements
- Ordinary-question rerank still uses a simple length-plus-keyword rule
- Conditional HyDE triggering still uses a heuristic v1 policy (`empty / low-score / weak-term-coverage`)
- Chunking is still basic and should later move to stronger semantic / structural chunking
- Evidence selection does not yet use a dedicated model-based reranker
- Grounded generation still uses a pragmatic `structured JSON first + markdown fallback` contract

V2 Roadmap Focus
- Smarter conditional HyDE triggering beyond today’s heuristic thresholds
- Better Chinese fuzzy-question retrieval and follow-up question handling
- Stronger document parsing for tables, headings, and field-like structures
- Richer evaluation and observability around retrieval hit quality and grounded answer usefulness
