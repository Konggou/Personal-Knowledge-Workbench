前台核心心智
- 项目 = 知识容器
- 会话 = 项目内对话线程
- 消息 = 主要交互单元
- 来源 = 回答底部的轻量证据层
- 知识库 = 资料管理页

V4 已落地
- 分支：`codex/v4-retrieval-optimization`
- 检索主链升级为：
  - SQLite `FTS5 + bm25`
  - Qdrant 语义检索
  - `RRF` 融合
  - 独立 reranker
- reranker 支持：
  - `rule`
  - `cross_encoder_local`
  - `cross_encoder_remote`
- second-pass 结果并入同一融合框架
- 增加 retrieval/index version 与自动 FTS 重建
- retrieval eval 新增：
  - lexical hit 数
  - semantic hit 数
  - fused hit 数
  - rerank backend / applied
- 修复了 V4 评审中发现的问题：
  - 空项目词法检索不再有跨项目泄漏风险
  - FTS 重建与计数改为只跟踪最新 snapshot
  - 规则 rerank 改用中文友好的 query terms builder

默认聊天时延优化
- 对普通项目内问答，如果没有开启：
  - `深度调研`
  - `联网补充`
  后端会优先走启发式 planner 与启发式 readiness
- 同时收紧 HyDE 触发，避免简单问题走不必要的 second pass
- 本地阶段性测试结果：
  - 简单问题总耗时约从 `36.15s` 降到 `4.50s`
  - 复杂问题总耗时约从 `45.56s` 降到 `6.44s`
- 当前主要瓶颈已经变成最终 grounded 回答的 LLM 生成，而不是 SQLite / Qdrant / RRF / 规则 rerank

项目删除与数据清理功能

- 新增测试覆盖：项目删除软删除验证、已归档项目访问验证、404 验证
- 修复：已归档项目的会话和来源不再出现在全局列表和项目列表中
