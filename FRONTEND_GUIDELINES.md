# 前端实现与设计指南

## 1. 目标

本文定义前端布局、视觉方向、交互规则与实现边界。

产品定义以 `PRD.md` 为准；本文不重复描述长期路线或后端细节。

## 2. 视觉方向

- 聊天优先
- Academic Editorial 深色主题
- 中间聊天区主导
- 暖金色强调
- 轻分隔线、少阴影、少并排大面板

核心颜色：

- Canvas: `#0a0c0f`
- Surface: `#111419`
- Text Primary: `#f0f0f0`
- Text Secondary: `#9ca3af`
- Accent: `#e6a845`

## 3. 页面布局

项目页结构固定为：

- 顶部轻导航
- 左侧项目树 sidebar
- 中间聊天主区

硬性规则：

1. 项目页不允许常驻右侧知识库栏
2. 项目页不允许大型项目头部
3. sidebar 展开时显示项目树；收起后必须缩成窄 rail，并让中间真实变宽
4. 中间消息列与底部输入框必须共用同一中轴
5. 摘要卡和报告卡必须留在会话里，不跳独立页面

## 4. 页面职责

### 4.1 `/workspace`

- 项目创建与进入入口
- 最近活跃项目列表
- 项目搜索与归档筛选

### 4.2 `/sessions`

- 全局会话浏览
- 按项目分组展示
- 支持重命名与删除

### 4.3 `/knowledge`

- 资料管理
- 按项目分组浏览与搜索
- 资料预览

### 4.4 `/settings`

- 全局模型与检索设置
- API Key 管理
- 保存反馈、校验与生效说明

### 4.5 `/projects/[projectId]`

- 主工作区
- 项目空状态
- 项目聊天与结果沉淀

## 5. 聊天页交互

### 5.1 项目空状态

- 未指定 `sessionId` 时，不自动打开会话
- 中间区域显示空状态，引导：
  - 新建会话
  - 前往知识库

### 5.2 会话态

- 聊天主区围绕当前项目的真实会话数据流
- 会话创建、切换、重命名、删除后，列表和主区要同步更新
- 不允许把全局 sessions 逻辑继续侵入项目主页面

### 5.3 输入区

输入区至少包含：

- `增加资料`
- 文本输入
- `深度调研`
- `联网补充`
- `生成报告`
- `发送`

规则：

- 所有动作使用轻量胶囊按钮
- `增加资料` 打开极小菜单，只包含文件和网页两类
- `深度调研` 与 `联网补充` 是会话内提问开关
- 开关状态应按会话记忆，而不是发送后立即丢失
- `生成报告` 默认基于当前会话最近一次有效结论

## 6. 来源层

来源层是回答下方的轻量证据界面。

固定交互闭环：

1. 先展示 `n 个来源`
2. 点击后展开来源标题列表
3. 再点击标题打开覆盖式详细预览
4. 支持返回来源列表

规则：

- 来源层只显示最终证据集
- 不展示召回候选、内部 rerank、task-like 过程
- 空来源时要给出明确解释
- 网页补充主导时，应能看出来源来自外部网页

## 7. 结果卡

### 7.1 摘要卡

- 作为会话内结果卡出现
- 应继承与最近有效结论一致的最终来源集

### 7.2 报告卡

- 仍留在当前会话流中
- 不跳独立报告页
- 若缺少有效结论，应阻止生成并给出清晰提示

## 8. 状态与错误

- 流式回答失败时，应保留用户问题与已生成内容
- 深度调研失败时，应留在当前会话中显示可恢复提示
- 联网补充失败时，应区分“网页补充失败”和“最终证据不足”
- 设置保存失败时，应保留表单值并给出明确反馈

## 9. 前端实现边界

- 不新增 task dashboard
- 不新增 asset center
- 不新增独立 search page
- 不把产品叙事改成 Agent 平台
- 不让知识库入口变成隐藏能力
