# 个人知识工作台 v1 前端规范

## 1. 设计目标

前端必须体现以下方向：

- 聊天优先
- 中间聊天区绝对主导
- 项目感清晰，但不做工作台卡片拼盘
- 知识库能力保留，但不抢主线程
- 来源可回看，但不切碎主聊天区

整体气质：

- 参考 Claude / Codex 这类聊天产品的空间分配
- 保留当前暖色底和柔和工作氛围
- 不做纯复制

## 3. 信息架构

一级导航固定为：

- `工作台`
- `会话`
- `知识库`

页面职责：

- `工作台`：唯一项目入口，创建项目并回到最近活跃项目
- `会话`：按项目分组浏览最近会话
- `知识库`：全局资料管理、搜索、预览
- `项目聊天页`：左侧 sidebar + 中间聊天主区

## 4. 项目聊天页布局

### 固定结构

- 顶部：轻导航，只承担全局切换
- 左侧：真正的 sidebar
- 中间：主聊天区

明确禁止：

- 项目页大型标题头部
- 常驻右侧知识库栏
- 多块大卡片并排布局

### 左侧 sidebar

必须展示：

- 全部项目树
- 当前项目默认展开
- 其它项目默认收起
- 每个项目下的会话列表
- 当前项目 / 当前会话高亮

收起规则：

- 收起后保留窄 rail
- 中间聊天区必须真实变宽
- 不能只隐藏内容，不改变布局列宽

### 中间聊天主区

主区必须优先像聊天产品，而不是卡片工作台。

结构规则：

- 外层吃满剩余宽度
- 内层消息列使用 `max-width`
- 消息列居中
- 输入框与消息列同宽、同中轴

## 5. 整体化表面规则

项目页必须看起来像一个连续界面。

因此：

- sidebar 与主聊天区主要靠轻分隔线区分
- 页面整体使用单一连续背景
- 不允许 sidebar、chat、composer、banner、message 共用“大圆角卡片外壳”

只允许在少量需要聚焦的地方保留表面层次，例如：

- 用户消息气泡
- 底部输入框
- 极轻状态提示
- 来源预览浮层

## 6. 消息样式规范

### 用户消息

- 靠右
- 轻气泡
- 视觉上清楚区别于助手内容

### 助手回答

- 靠左
- 正文流排版
- 弱化边框和大块盒子感

### 状态卡

- 轻量
- 只表达关键阶段，如：
  - 调研中
  - 调研完成
- 不做工程时间线

### 来源更新卡

- 比普通回答更轻
- 明确提示资料已加入当前项目

### 摘要卡 / 报告卡

- 作为会话内独立结果卡出现
- 只保留 `复制 / 删除`

## 7. 输入区规范

输入区固定在聊天主区底部。

必须包含：

- `增加资料`
- 文本输入
- `深度调研`
- `生成报告`
- `发送`

规则固定：

- `知识库` 小入口放在输入框上方
- `增加资料 / 深度调研 / 生成报告` 使用轻量文本按钮或小胶囊
- 不允许再出现大块工具按钮
- `增加资料` 点击后弹出极小菜单：
  - 添加文件
  - 添加网页链接
- 选择网页链接后，在输入区上方出现小型 URL 表单

## 8. 来源层规范

第一层：

- 回答底部显示 `n 个来源`
- 最多显示 3 个来源图标

第二层：

- 点击来源气泡后，在当前消息下展开轻量来源标题列表
- 第一层不显示摘录

第三层：

- 点击来源标题后，打开覆盖式来源预览浮层
- 不再通过常驻右侧栏承载

## 9. 文案规范

统一使用以下术语：

- 项目
- 会话
- 知识库
- 来源
- 深度调研
- 生成报告
- 保存为摘要
- 弱资料模式

前台主文案禁止重新引入：

- task
- asset
- search page
- task detail
